## Contribution types

You can contribute by:
- Reporting bugs
- Improving documentation
- Adding filters or strategies
- Improving test coverage
- Refactoring internal code

---

## Before submitting a PR

Please ensure:
- Tests pass locally
- Documentation is updated if behavior changes
- Code follows existing style and patterns

---

## Pull request guidelines

- One logical change per PR
- Clear commit messages
- Descriptive PR description
- Reference related issues if applicable

---

## Code style

- Keep functions small
- Prefer explicit errors
- Avoid side effects in plugins
- Use structured logging (`slog`)

---

## Breaking changes

Breaking changes must:
- Be clearly documented
- Be discussed before merging
- Update both user and dev documentation

---

## Code of conduct

Be respectful, constructive, and patient.
